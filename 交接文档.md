## 2. 当前产品定位与不可变约束

PromptMate 是面向新手和轻度创作者的 Windows 10/11 本地提示词助手。核心是提示词词库的浏览、发现、筛选、收藏、组合、复制和本地管理；AI、图片转提示词与视频转提示词是可选辅助。产品不直接生成图片或视频，不依赖用户账号、自建业务服务器、云同步或远程配置中心。

当前已确认的后续产品决策：

- 顶层界面只区分图片提示词和视频提示词；
- 不按生成平台或目标生成模型拆分词库，也不要求用户先选平台/模型；
- AI 服务模型名只用于用户所配置的文本/视觉服务，不得影响词库分类；
- 用户词条表单只有中英文名称和分类必填，描述、标签、别名均选填；
- AI 模型必须从服务商官方 `/models` 同步并由用户明确选择，同时保留手动输入退化路径；
- 图片提示词资料中的全部可识别概念都要保留，包括成人、敏感、学校、年龄和争议内容；
- 不建立 R18 专区，所有内容按真实语义分类；
- 每个独立功能必须测试、独立复审、单独 Conventional Commit、push，并从最终提交重建和启动 Release EXE。

## 4. 小分类体系：当前最高优先级

当前顶部筛选只是 `新手友好 / 人像 / 电影感 / 东方美学 / 自然 / 商业 / 成人向 / 截图整理 / YouMind整理` 等横向标签，不是原始方案要求的完整二级分类。

### 4.1 目标数据模型

- 每条词条必须保留一个父 `category_id`；
- 评估新增可选 `subcategory_id`，继续复用现有 `PromptConcept` schema，不创建第二套数据格式；
- `subcategory_id` 表示父类内部语义细分；
- 来源、成人向、使用场景、审美、受众等使用独立 Facet/标签，不与语义小分类混合；
- 用户分类需支持新增、移动、隐藏、删除和迁移；内置标准分类可隐藏但不可删除；
- 无法确定的小分类进入“待分类”，不能随意猜测；
- 删除用户分类前必须选择迁移内容或一并删除。

### 4.2 与原始分类体系的映射

原始完整方案要求的 20 类图片标准分类，当前被压缩为 11 个父类。建议通过小分类恢复其表达力：

| 当前父类 | 需恢复的原始细分 |
|---|---|
| 人物主体 | 主体与人物种类；性别、年龄与身份；外貌、发型与妆容 |
| 服装配饰 | 服装与配饰；材质、图案和穿着状态 |
| 动作姿态 | 动作与姿态；人物互动；手部/腿部/视线行为 |
| 表情与情绪 | 表情与视线；情绪和特殊面部状态 |
| 场景环境 | 场景与环境；背景元素；季节、天气、时间、节日 |
| 物品与道具 | 道具与背景元素；家具、电子设备、武器、交通工具等 |
| 灯光氛围 | 灯光；雾、光束和大气光学现象 |
| 镜头构图 | 机位与视角；景别与镜头；构图；分辨率与画幅中的画幅部分 |
| 艺术风格 | 艺术与视觉风格；绘画媒介；地域风格；设计排版 |
| 画质与特效 | 色彩与调色；滤镜；特效；材质与细节；画质；分辨率 |
| 负面提示词 | 画质、人体、构图、文字水印、内容与风格排除 |

建议小分类候选：

- 人物主体：人数、性别/年龄角色、职业身份、物种、身体部位、发型、妆容、角色原型；
- 服装配饰：上装、下装、裙装/制服、内衣/泳装、袜类、鞋履、头饰、首饰、材质/图案、穿着状态；
- 动作姿态：站、坐、跪/蹲、躺、手部、腿部、视线、互动、运动/战斗、成人互动；
- 表情与情绪：笑、哭、愤怒/厌恶、恐惧/紧张、害羞/尴尬、困惑/思考、疲惫/无表情、特殊面部状态；
- 场景环境：自然、城市、建筑、室内、季节/天气、时间、节日、水域、幻想/科幻、背景类型；
- 物品与道具：家具、电子、餐具/食物、武器、交通工具、乐器、文具、玩具、卫浴、成人道具；
- 灯光氛围：自然光、轮廓光、体积光、霓虹、电影光、背景光、镜头光效、雾/大气；
- 镜头构图：景别、视角、主体朝向、裁切、焦点、透视、自拍/镜面、多格/分屏、留白/版式、画幅；
- 艺术风格：摄影/写实、绘画媒介、动漫漫画、复古、传统地域艺术、三维/手工、平面/矢量、设计排版；
- 画质与特效：分辨率/质量、色彩/单色、材质光泽、粒子/辉光、AR/全息、运动表现、格式/后期；
- 负面提示词：画质缺陷、人体结构、手脚、构图裁切、文字水印、主体数量、风格排除、内容排除。

### 4.3 实施顺序

1. 导出每个父类的完整词条清单、词频和候选语义簇；
2. 人工确认小分类名称、定义、排序、边界和示例；
3. 增加 schema 与 `libraryCatalog.ts` 元数据；
4. 建立 `id → parent → subcategory → facets` 迁移台账；
5. 先迁移人物主体与服装配饰；
6. 写 RED 测试锁定注册完整性、代表词条、无孤儿小分类和筛选状态；
7. 独立复审并单独 commit/push/Release；
8. 用户验收后再迁移其余父类；
9. 完成后增加用户自定义分类/子分类管理。

## 7. 原始完整方案中被遗漏的大功能

### 7.1 悬浮窗与自动粘贴（M8，未开始）

原始完整方案明确要求：

- 可拖动、置顶、吸边和贴边收起的悬浮球；
- `智能优化` 与 `词库快选` 两个悬浮页签；
- 全局快捷键，可修改或禁用；
- 记录前台窗口和焦点控件；
- 在目标输入框光标处追加提示词；
- 按组合规则决定分隔符；
- 撤销上一次插入；
- 中文/Unicode 使用临时剪贴板粘贴，并尝试恢复原剪贴板；
- 权限、焦点或受保护控件失败时退回复制并提示；
- 自动粘贴、剪贴板恢复、开机启动、置顶均为独立开关；
- 普通权限进程不得向管理员窗口注入输入。

当前仓库没有 `WebviewWindow` 悬浮窗或 Windows 自动化模块。这不是已取消需求；需由产品确认是否仍进入第一版。若保留，应作为独立高风险里程碑，不与小分类混在同一提交。

### 7.2 Embedding 与跨语言语义检索（M6，未开始）

原始要求包括：

- 本地/远程 Embedding 分开配置；
- Unicode、精确、别名、关键词和向量混合检索；
- 按分类、媒体、适用性加权；
- 高置信复用、临界文本模型复核、低置信生成候选；
- `低机位 / 仰拍 / low-angle shot` 等跨语言评测组；
- 阈值配置化，不能凭感觉写死；
- SQLite 或本地向量索引；
- 未安装本地模型时降级为词法并说明差异。

当前只有词法/别名搜索和 AI 配置中的 embedding 能力占位，没有实际 Embedding 适配器、索引或评测集。

### 7.3 模型档案与组合规则

原始方案曾要求平台/模型两级选择和多模型优化；后续产品决策覆盖了面向用户的平台/目标模型选择，当前不应恢复平台化顶层 UI。但以下数据驱动能力仍可保留为**媒体组合规则**：

- 图片/视频字段顺序；
- 中英文分隔符；
- 负面提示词；
- 视频时序结构；
- 用户复制、编辑、导入和导出规则；
- 通用图片与通用视频兜底。

“针对 Midjourney/FLUX/Sora 等目标模型分别优化”和“硬编码平台模型列表”属于**后续决策覆盖**，除非产品再次明确恢复，不应擅自实施。

### 7.4 Bing 解释与资料搜索

原始 MVP 明确要求使用系统浏览器打开经过 URL 编码的 Bing 网页/图片搜索。当前没有正式 `https://www.bing.com/search?q=...` 实现，需补充：

- 词条卡片/详情中的解释或搜索入口；
- 中文、英文、空格、符号和 Unicode URL 编码测试；
- 系统默认浏览器打开；
- 离线时仍可生成 URL，但需明确网络依赖。

### 7.5 数据来源链和第三方数据文件

原始完整方案要求每条数据保留仓库、文件、commit、作者、许可证、URL、转换步骤和导入时间，并自动生成 `THIRD_PARTY_DATA.md`、NOTICE 和来源索引。当前数据只做到 README 和部分描述字段追溯；完整 provenance 尚未落地。

用户已接受“提示词来源都是网络收集而来”作为当前阶段说明，但这不等于原始来源链需求已完成。二者要区分：当前技术交付不以逐条许可阻断，未来公开再分发仍需来源/权利链工作。

## 8. 被后续产品决策覆盖的原始需求

以下原始描述不再是当前默认路线：

- 第一版必须提供平台和具体目标生成模型选择；
- 针对 Midjourney、FLUX、Sora、Veo 等建立用户可见的模型专属词库入口；
- 平台/目标模型决定顶层分类；
- 第一版同时完整实现所有 M0–M10；
- AI 自动生成缺失词条后不经确认直接进入正式词库；
- 产品直接执行图片或视频生成任务。

当前替代决策：

- 只保留图片/视频顶层；
- 通用词库和通用组合优先；
- AI 服务模型由用户配置，仅用于可选文本/视觉辅助；
- AI 新词条应进入待确认流程（仍需最终产品确认）；
- 不直接生成图片/视频。

## 12. 开发与交付约束

- 一个独立功能一个 Conventional Commit；
- 每个大功能先测试、代码审查和 Bug 检查，再 commit/push；
- 复审绑定精确 `git write-tree`，任何修改都会使旧审查失效；
- 桌面可见功能必须从最终提交重建并启动同一路径 Release EXE；
- 不提交原始截图、OCR JSON、临时脚本、清洗报告或虚拟环境；
- 不删除用户资料中的真实提示词，不建立 R18 专区；
- 不在无真实工具证据时声称测试、构建、push 或 Release 成功；
- 中文仓库路径的 shell 命令从父目录进入：

```bash
cd '提示词助手' && <command>
```
